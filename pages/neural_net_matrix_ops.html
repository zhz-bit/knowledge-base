◆ 反向传播之前,神经网络怎么"优化"?

1986 反向传播普及之前,神经网络主要靠下面这些方式"优化"——它们大多只能训单层或局部,正是这个局限引发了第一次 AI 寒冬。

核心瓶颈:信用分配问题(隐藏层没有"标准答案")

多层网络卡了二十年,根子在信用分配(credit assignment):隐藏层的神经元没有"标准答案",你不知道该把多少误差算到它头上 —— 所以早期规则都只能优化输出层 / 单层。Minsky & Papert《Perceptrons》(1969)指出单层连 XOR 都解不了,而当时又训不了多层 → 研究停滞。

反向传播解决的正是这一点:用链式法则把输出端的误差逐层回传,给每个隐藏层权重都算出梯度。点按钮切换"前向算预测"和"反向传误差"两步:

反向传播之前的几种"优化"

反向传播的意义有多大?

关键不在"用链式法则求导"本身,而在效率:

  • 一次反向传播,同时算出所有参数的梯度,代价和一次前向差不多(O(网络规模))。
  • 对照:数值微分要为每个参数单独扰动跑一遍前向 → O(参数量 × 前向),对上亿参数天文数字、根本不可行。
  • 通用:MLP / CNN / RNN / Transformer / GNN 全靠它训练;PyTorch / TensorFlow 的自动微分本质就是它。
  • 底座的底座:Adam、归一化、初始化……都建立在"反向传播能廉价给出梯度"之上。
一句话:没有反向传播,就没有任何能训练的深网。它是整个领域唯一的训练引擎。

那为什么过了二十年才流行?(其实早被想到、却被埋没)

反向传播并非没人想到 —— Linnainmaa(1970,反向模式自动微分)、Werbos(1974 博士论文用于神经网络)早已提出。它沉睡多年,是因为:

原因说明
领域已放弃神经网络1969《Perceptrons》之后连接主义被边缘化、经费枯竭,没人在找"怎么训多层"
算力太弱70 年代计算机即便用它也跑不出像样结果,说服不了人
理论悲观普遍认为多层非凸、会卡在局部极小,"算了也没用"
缺演示 + 对的圈子Werbos 的工作在控制 / 经济学语境;直到 1986 Rumelhart 等在认知科学(PDP)圈、配上"学出隐藏层表示、解 XOR"的实验,才真正点着
事后看它"显然"(无非链式法则),但当时的寒冬 + 悲观 + 弱算力 + 错圈子,让它沉睡了十几年 —— 直到 1986 被讲清、被验证,深度学习才真正起步。
附:Hinton 的 2024 诺贝尔物理奖是因玻尔兹曼机(用统计物理),不是因反向传播;反向传播也非他一人发明。

① 先看懂一个方块代表什么

所有网络层的核心其实只有一件事：矩阵乘法（再加个偏置、过个激活函数）。

一个矩阵就是一张数表。规则只有一条 —— “左边的行” 点乘 “右边的列”：

输出[ i , j ]　=　 Σk　A[ i , k ] × B[ k , j ]

下面 A (2×3) 乘 B (3×2) 得到 C (2×2)。点“下一步”，看高亮的一行和一列如何配对相乘、再相加填进输出格。

当前行（A） 当前列（B） 算出的输出格
记住一个尺寸规则： (m×n)·(n×p) = (m×p)。中间那个 n 必须相等（行长 = 列高），结果的形状取两头。后面所有网络都在反复用这条。

② 全连接层（MLP / Dense / Linear）

最朴素的一层：每个输出神经元，都连到所有输入。

y = W x + b　　然后 a = σ(y)，σ 是激活函数

设输入是 4 维向量 x，输出是 3 维向量 y。那么权重 W 就是一张 3×4 的表：第 i 行专门负责“怎么把 4 个输入加权成第 i 个输出”。

点“下一步”，逐个算出 3 个输出神经元。W 的第 i 行 与 整列 x 做点积，加上偏置 bi，就是 yi。

本质： 一层 MLP = 一次矩阵乘 + 一次加偏置。“全连接” 三个字，对应的就是 W 里没有一个 0 —— 每个输出都看了全部输入。参数量 = 3×4 + 3 = 15。记住这个“稠密、看全局”的特点，第⑥节对比 CNN 时要用。

③ CNN（卷积）

卷积看起来像“小窗口在图上滑动”，但底层一样是一次大矩阵乘法。

绿 = 输入 / 激活值（输出也是激活，仍为绿） 蓝 = 卷积核权重 深浅 = 不同通道

一张 4×4 的输入，用一个 2×2 的卷积核（kernel）去扫。核停在每个位置，就把它盖住的 4 个数和核的 4 个权重对应相乘再求和，得到输出图上的一个数。滑过整张图，得到 3×3 输出。

第一步：核在滑动

点“滑动一格”，看绿色窗口怎么移动，以及每一步算出的输出。

第二步：把卷积“摊平”成矩阵乘法（im2col）

计算机其实不会真的“滑动”。它把每个窗口盖住的 4 个数抽出来拉成一行，9 个窗口堆成一个 9×4 的大矩阵（叫 im2col）；卷积核拉成 4×1 的列。两者一乘，9×4 · 4×1 = 9×1，再 reshape 回 3×3 —— 卷积就变成了一次和 MLP 一模一样的矩阵乘。

⊙ 还是 · ？为什么"一步看像逐元素积，整体却是矩阵乘"

这正是最容易混的地方。盯着一个窗口时，patch 和 kernel 都是 2×2、摆正对齐，"对应位置相乘"看起来就是哈达玛积（⊙，逐元素积）。但别忘了后面还有一步：把 4 个乘积全部加起来，塌成一个数。"⊙ 之后整体求和"——把两个 2×2 都拉直成向量后，恰好就是一次点积（内积），而点积正是矩阵乘法里"一个格子"的算法。

所以卷积并没有"变成"哈达玛积： 单个位置是 ⊙ + 求和 = 点积 = 矩阵乘里的一个格子；把所有位置打包起来，就是一次完整矩阵乘法。滑动视角只看一个格子(于是看到 ⊙)，im2col 视角一次看全部格子(于是看到 matmul)，是同一运算的两种粒度。提醒：纯哈达玛积 A⊙B 是不求和的、结果仍是同形状矩阵 —— 卷积比它多了"求和塌成标量"这一步，正是这一步让它等于点积、从而能拼成矩阵乘法。
两个关键不同（相对全连接）：
① 局部连接：每个输出只看一个 2×2 的小邻域，不是整张图。
② 权重共享：9 个窗口用的是同一套 4 个权重。所以一个卷积核只有 4 个参数，而不是 16×9 个。这正是 CNN 参数少、又能抓“平移不变”特征的原因。

④ RNN（循环网络）

处理序列。诀窍是：同一套权重，沿时间一遍遍复用，并把“记忆” h 往后传。

ht = tanh( Wxh xt + Whh ht-1 + b )

每个时间步 t 做两次矩阵乘再相加：
· Wxh · xt：当前这一帧输入带来的信息；
· Whh · ht-1：上一时刻的隐藏状态（记忆）带来的信息。
两者相加、过 tanh，得到新的记忆 ht，再传给下一步。

点“前进一个时间步”，看记忆 h 如何从左向右流动。注意：Wxh 和 Whh 在每一步都是同一张表（又一次权重共享，只不过这次是沿“时间”共享）。

和 CNN 的呼应： CNN 沿空间共享权重，RNN 沿时间共享权重。两者都是把“一个小 MLP（W）”反复套用到不同位置/时刻上。

✦ 液态神经网络：LTC / NCP

RNN 谱系里很别致的一支 —— 把"记忆快慢"做成随输入流动的连续时间动力学,再配上仿线虫的稀疏接线。

① LTC 神经元：时间常数会"流动"

上面的 RNN 是离散时间、动态固定。推到连续时间就是 CT-RNN / Neural ODE:dx/dt = −x/τ + f(x,I),τ 是固定的时间常数(决定遗忘/记忆的快慢)。LTC(Liquid Time-Constant,Hasani 等,AAAI 2021)把 τ 变成随输入变化的:

dx/dt = −[ 1/τ + f(x, I) ]·x + f(x, I)·A

方括号里的 1/τ + f(x,I) 就是"液态"的有效时间常数 —— 输入越强,神经元响应/遗忘越快。灵感来自线虫(C. elegans)神经元的非线性突触动力学,还能证明稳定(状态与时间常数都有界),且比固定 τ 更会拟合时间序列。拖滑块看 τ_sys 随输入变(琥珀=LTC 自适应,灰=固定 τ 的普通 RNN):

② NCP 网络：用 LTC 神经元搭一张"线虫接线图"

NCP(Neural Circuit Policies,Lechner 等,Nature MI 2020) = 一批 LTC 神经元 + 一张稀疏、仿线虫连接组的接线。经典的四层结构(连接很稀疏,每个神经元只接几根):

感觉 → 中间 → 指令(有循环连接=记忆)→ 运动。惊人结果:约 19 个控制神经元 + 一个卷积感知头,就能端到端开自动驾驶车(车道保持);而且网络小到能逐个神经元看懂,做到了"可审计的自治"。

串到本页主线: RNN 沿时间共享权重;LTC 把它升级成连续时间 + 输入自适应的时间常数(更稳、更省、更像生物);NCP 再配上稀疏结构先验(接线刻进网络形状)。又是"同一种矩阵乘法 + 不同结构约束"的一例:用极少神经元换来鲁棒与可解释。后续 CfC(2022)还把 LTC 的 ODE 求解写成闭式,跑得更快。

◈ 连续视角:ResNet = 欧拉法 → Neural ODE

上一节的 LTC 已经是个"ODE 神经元"。其实整套思路只有一句:把层数 / 时间从离散变连续,让网络拟合变化率(导数),再用 ODE 求解器积分。

关键观察:残差网络的一层 h + f(h) 正好是欧拉法解微分方程的一步(ht+Δ = ht + Δ·f)。堆 N 层残差 = 用欧拉法把 ODE 积分 N 步。把 N 推到连续,就是 Neural ODE(Chen 等,2018):dh/dt = fθ(h, t),前向 = 调一个 ODE 求解器。

欧拉法:看斜率 → 走一步(微分方程怎么"解")

微分方程只告诉你每一点的斜率 dy/dt = f(y,t),没直接给曲线;欧拉法就顺着斜率一小步步走,把曲线拼成折线。下图淡蓝细线 = 斜率场(每点该往哪个方向走),蓝曲线 = 真解。点"走一步",看它在当前点看一眼斜率(黄箭头)、沿它迈 Δt 到新点:

N 步欧拉 = N 层 ResNet,N→∞ = Neural ODE

每一步 y + Δt·f(y) 就是一层残差。把步数 N 一路调大:N 小时折线糙、偏离真解;N 越大越逼近那条光滑曲线 —— N→∞ 就是连续的 Neural ODE。

ODE 在神经网络里的四个落点(其实都在本页图谱里)

落点形式对应
连续深度dh/dt = f(h),深度=时间ResNet(2015)→ Neural ODE(2018)
连续时间序列dh/dt = −h/τ + …,真实时间LTC / 液态网络、ODE-RNN —— 处理不规则采样
状态空间 SSM线性 ODE dh/dt = Ah + Bx 离散化Mamba / S4
连续生成流学速度场 dx/dt = v(x,t),积分即生成扩散概率流 ODE、Flow Matching / MeanFlow
怎么训练它:用伴随法(adjoint) —— 反向再解一个伴随 ODE 倒着积分回起点,就能拿到梯度,而且 O(1) 显存(不存中间每一层)。
一句话:ODE 融进神经网络,就是"让网络拟合一个导数场 dh/dt = fθ,再交给数值积分器"。残差、液态、Mamba、流匹配看似四件事,底下是同一条"连续模型"的线。

拓展:微分方程 —— 常微分(ODE)与偏微分(PDE)

先说"微分方程"是什么。普通方程(x+3=7)解出来是一个数;微分方程的未知量是一个函数,它给的是"函数和它自己的变化率(导数)之间的关系"。为什么有用?——自然界往往只给我们"变化的规律",不直接给结果:"冷却速度正比于温差""加速度 = 力 / 质量""人口增长正比于现有人口"……这些都是关于导数的规则。"解"微分方程,就是从"变化的规则"反推出"那个实际的函数"。

① 常微分 ODE:未知函数只依赖一个变量

它和普通方程差在哪、又解决什么问题?普通方程 x+3=7 的未知量是一个数 → 解是一个点(x=4);常微分方程的未知量是一个函数 → 解是一整条曲线。它解决的处境是:我们只知道"变化的规律(速率)",却不知道量本身 —— 给定规律 + 一个起点,反推出整条轨迹。

拿牛顿第二定律(F = ma)举个具体例子:一个下落的物体。重力 mg 往下拉(让它加速),空气阻力 k·v 往上顶(速度越快阻力越大)。把 F = ma 写出来,就是一个 ODE:

m·(dv/dt) = mg − k·v　⟹　dv/dt = (k/m)·(v终 − v)

它只告诉你"此刻速度变化多快"(=加速度),没直接给速度。解这个 ODE 得到的,就是速度随时间的曲线 v(t):一开始重力占上风、加速;速度越大阻力越大,直到两者抵消(加速度=0),速度稳在终速 v终 = mg/k(就是跳伞员的"终端速度")。

所以"点选的曲线"到底是什么?就是这条 v(t)。每点一下 = 在某时刻给物体一个初始速度,画布就画出它之后速度随时间的真实轨迹(= 这条 ODE 的解)。细箭头是规律(每点 dv/dt 朝哪),曲线就是顺着箭头一路走出来的。不同初速 → 不同曲线,但都收敛到同一个终速。

"常"= 未知函数只依赖一个自变量(通常是时间 t),式子里只有普通导数 dy/dt。规则 dy/dt = f(y,t) 给出每一点的斜率,再加一个初始条件 y(0)=y₀ 钉住"是哪条解"。从简单到不简单:

  • dy/dt = k:匀速变化 → 直线 y = y₀ + kt;
  • dy/dt = k·y:变化正比于当前量 → 指数 y = y₀·ekt(人口、放射衰变、复利);
  • d²x/dt² = −k·x:二阶 → 来回振荡 sin/cos(弹簧、单摆)。

几何上就是前面那个斜率场:每点一个方向箭头,解 = 从起点顺着箭头走出的一条曲线。→ 一个状态,沿一条轴演化。

② 偏微分 PDE:未知函数依赖两个以上变量

当未知量是 u(x,t)(同时依赖位置 x 和时间 t),求导只能"对其中一个变量、按住其余"——这就叫偏导 ∂u/∂t、∂u/∂x。它把"随时间的变化"和"随空间的变化"联系起来。最经典的热传导方程:

∂u/∂t = α · ∂²u/∂x²

读法:某点升温 / 降温的速度(∂u/∂t)正比于它周围温度的"弯曲程度"(∂²u/∂x²)。比邻居平均更冷(处于"低谷",∂²u/∂x²>0)就升温,更热就降温 → 温度被不断抹平 = 扩散。PDE 除了初始条件(t=0 时整片场长啥样),还需要边界条件(杆两端怎么处理),因为空间是有边的。→ 一整片分布,随空间 + 时间演化。

⑤ Transformer（自注意力）

不再逐步循环，而是让序列里每个词同时看所有词，用一张“注意力分数表”决定该关注谁。

输入是 序列长度 × 维度 的矩阵 X（这里 3 个词、每个 4 维）。先用三张投影矩阵把它变成 Query / Key / Value：

Q=X·WQ　　K=X·WK　　V=X·WV　（三次 MLP 式的矩阵乘）

然后是注意力的核心公式：

Attention = softmax( Q Kᵀ ⁄ √d ) · V

Q · Kᵀ（3×4 · 4×3 = 3×3）：让每个词的 Query 和每个词的 Key 做点积，得到一张“谁该关注谁”的相关性分数表。每行经过 softmax 变成一组和为 1 的权重，再去对 V 加权求和，就得到融合了上下文的新表示。

点“下一步”，依次看：① 算出 Q、K、V　② Q·Kᵀ 得到注意力分数表　③ softmax 归一化　④ 乘 V 得到输出。

一句话： Transformer 把所有“关系”都压进了一次 Q·Kᵀ 的矩阵乘里 —— 这就是它能并行、能抓长距离依赖、又特别吃 GPU（矩阵乘最擅长的活）的根本原因。

深入：QKᵀ 能合并成一个矩阵吗？

一个很自然的疑问：Q·Kᵀ = X·(W_Q·W_Kᵀ)·Xᵀ，那把 W_Q·W_Kᵀ 合并成一个矩阵 M 不就行了？数学上完全成立——但实际不这么做。展开看为什么:

▸ 展开：合并恒等式、低秩瓶颈、与 QK circuit
Q Kᵀ = (XW_Q)(XW_K)ᵀ = X·(W_Q W_Kᵀ)·Xᵀ = X·M·Xᵀ　(M = W_Q W_Kᵀ, d×d, 秩 ≤ d_k)
先说清 d 和 d_k(两个维度)：
· d = 模型维度（文献里常写作 d_model，和这里的 d 是同一个东西，别被两种写法搞混）：每个 token 向量的长度(词嵌入维)。输入 X 是 序列长 n × d。
· d_k = 每个注意力头里 Q/K 的维度。多头时 d_k = d / h(d 被 h 个头平分)。投影矩阵 W_Q、W_K 形状 d×d_k，作用是把每个 d 维 token 压成 d_k 维的 query / key(所以 Q=X·W_Q 是 n×d_k)。
· 例:上面 ⑤ 的演示里 d=4、d_k=2(W_Q 就是 4×2)。下面滑块为看得清取 d=12，拖 h 看 d_k=d/h 怎么缩、M 怎么变低秩。

对固定模型,这步严格相等——scores 完全一样,所以纯是参数化方式之争。拖滑块感受核心那条理由(低秩瓶颈):头数 h 越大,每头 d_k=d/h 越小,M 越低秩。

三个不合并的理由：
① 低秩瓶颈(最主要)： M=W_QW_Kᵀ 是 d×d 但秩 ≤ d_k=d/h(多头里 d_k≪d)。两个 d×d_k 矩阵是 M 的低秩分解;合并成满秩 M,参数从 2·d·d_k 涨到 d²。这个瓶颈是特意的归纳偏置+正则,也是多头省参的根本。
② 计算更贵： 分解让 Q、K 活在更小的 d_k 空间,投影和 n² 打分都便宜约 h 倍;合并反把它们拽回 d 空间。连推理都不会合并。
③ 多头： 每头各有低秩 M⁽ʰ⁾,压不成一个矩阵。
(注意:实现里把 W_Q/W_K/W_V 拼成 d×3d 做一次 matmul 是另一回事——只是并投影省 kernel,Q、K 仍分开,没把 W_QW_Kᵀ 乘成 M。)
优化上的区别(你问的重点)： 即使把 M 限成同样的秩、表达同一函数,训练 (W_Q,W_K) ≠ 训练 M:
· 损失对 (W_Q,W_K) 是矩阵乘积、非凸(有鞍点),对 M 则线性;
· 梯度互为预条件:∂L/∂W_Q = (∂L/∂M)·W_K(反之亦然),等价 M 上的有效梯度被这套耦合动力学重塑;
· 自带隐式低秩偏好(深度矩阵分解的 implicit bias)+ 显式 rank≤d_k 约束 = 正则,泛化常更好;
· 有规范冗余:W_Q→W_QA、W_K→W_KA⁻ᵀ 给同一个 M(过参数化,通常利于优化)。
漂亮的呼应 —— QK circuit： Anthropic《A Mathematical Framework for Transformer Circuits》里,做理解/可解释性时人们确实把 W_QW_Kᵀ 当一个双线性矩阵分析(称 QK circuit,同理 W_OW_V 是 OV circuit);但做计算和训练时保持分解。"理解时合并,计算时分解"——正是这个问题的最佳注脚。

深入：rank(M)=d_k 的夹逼 + 乘法压秩 / 加法升秩

承接上面。借矩阵的秩 = 像空间(列空间)的维数这把尺子,把 rank(M)=d_k 夹出来,并讲清"乘法只压秩、加法才升秩"。(这套线代基础已单独整理成一页:线性代数公共基础 → 含矩阵变换、秩↔特征向量的交互讲解。)

▸ 展开：秩的几何 + 夹逼定理 + 乘法/加法的秩

① 乘法压秩 —— 像穿不过比它窄的瓶颈。 M=W_Q W_Kᵀ 把向量先压进 d_k 维"细腰",再展开:

v ∈ ℝd —— W_Kᵀ ——▶ ℝd_k(细腰) —— W_Q ——▶ ℝd

两步定界:im(AB) ⊆ im(A) → rank ≤ rank A;AB 只作用在 im(B) 上 → rank ≤ rank B。合起来 rank(AB) ≤ min(rank A, rank B)。乘法绝不升秩——像空间穿不过比它窄的瓶颈,所以 M 名义 d×d、秩却卡在 d_k。

② 加法升秩 —— 两个像空间的"和"。 (A+B)v = Av+Bv,落在 im(A)+im(B) 里;两个子空间的和,维数 ≤ 各自维数之和: rank(A+B) ≤ rank A + rank B。加法能升秩(最多到两者之和)。

③ 夹逼出 rank(M)=d_k。 上界 = 乘法压秩 = min = d_k;下界 = Sylvester 秩不等式 rank W_Q + rank W_Kᵀ − d_k = 2d_k − d_k = d_k(当 W_Q、W_K 列满秩,随机/训练后几乎必然)。两边都 d_k,夹死:

d_k (下界·Sylvester) ≤ rank(M) ≤ d_k (上界·min) ⟹ rank(M) = d_k
合起来 = 多头的代数动机： 单头里 M=W_QW_Kᵀ 是乘法,秩被压死在 d_k;想升秩不能靠加大 d_k(>d 纯浪费),只能靠加法 —— 多头输出 Σ 各头贡献,h 个 rank-d_k 相加 → 总秩可达 h·d_k=d。乘法压秩 ⇒ 单头低秩;加法升秩 ⇒ 多头补回 d。 这就是"低秩瓶颈 + 多头"在线性代数上的真正动机。

从 MLP 到 CNN：一次彻底的拆解

上半场是"每种网络在乘什么"的巡览；下半场只做一件事 —— 把 CNN 和 MLP 的关系彻底讲透：从"CNN 是受限的 MLP"，到 LCN / Tiled 的完整光谱，再到多通道与"为什么偏偏是相加"。

A 受限的 MLP　·　B 一根光谱　·　C 两条约束　·　D 多通道　·　E 延伸

A · CNN 其实是受限的 MLP

可以，而且这是个很有洞察力的看法 —— 但要加两个限定词。

答案：能。 一个卷积核，本质上就是同一个小型全连接层（MLP），被复制贴到图像的每一个局部窗口上。把图“摊平”成 im2col 之后，卷积和 MLP 的计算完全是同一次矩阵乘法（见第③节）。所以：

卷积层　≈　一个小 MLP　×　在所有位置上重复使用（权重共享）　+　只连局部（局部连接）

全连接 MLP（稠密）

每个输出连全部输入；每条连接都有独立权重。

卷积 = 受约束的 MLP

把它的 W 加两条规矩，就变成卷积：

右边其实就是左边那张 W，只不过被强行加了两条约束：

角度全连接 MLPCNN（卷积）
连接范围每个输出看整张图（全局、稠密）每个输出只看一个局部窗口（稀疏）
权重每条连接各不相同所有位置共享同一套核权重
等价说法—把一个小 MLP 滑遍全图，W 是一个稀疏 + 重复的“块状”矩阵
参数量(本页例子)16→? 稠密，参数随输入暴涨一个核仅 4 个参数，与图大小无关
带来的好处表达力最强，但易过拟合、算不动大图参数少、平移不变、适合图像
那 PyTorch 里到底变换谁？——变换 img，不是变换核。
上面这张"稀疏 + 共享"的 W（把卷积核铺成一个巨大的稀疏 Toeplitz 矩阵、让图保持摊平）是数学等价的教学视角，专门用来理解"conv = 受限 MLP"，但实际不这么算——那个矩阵巨大且几乎全是 0，乘一堆零纯属浪费内存和算力。
真正的实现（就是第③节的 im2col）反过来：把输入图展开成 im2col 矩阵（torch.nn.Unfold / F.unfold，这步会复制数据），而卷积核只做一次廉价的 reshape（view）、权重本身不变、不复制，最后一次稠密 GEMM 搞定。
原因：权重共享意味着核要被复用到所有位置，所以核保持一份、小而密，要被"摊开铺满"的自然是图——展开图、而非展开核，才能把这种复用喂给 GPU 上最快的稠密矩阵乘。
但有两点不能忽略，否则会说反：
① 不是“多个不同的 MLP 拼接”，而是“同一个 MLP 被复用很多次”。 9 个输出位置共用一套权重，不是 9 个独立的全连接层。
② 反过来要分情况： 卷积（指小核滑动那种）是 MLP 的特例 / 受限版（加了局部 + 共享约束）。任何卷积都能写成一个稀疏、参数绑定的全连接；反过来，任意全连接也能写成一个核 = 整图的卷积（见下方"反过来想"），只是写不成小核滑动的那种卷积。

所以更精确的一句话是 —— “卷积 = 一个共享权重的小 MLP，在输入上滑动复用”； 而当 kernel 大到覆盖整张图、且每个位置不再共享权重时，CNN 就退化回了普通 MLP。这条线索其实贯穿全篇：CNN 沿空间复用、RNN 沿时间复用、Transformer 把复用换成”一次性两两点积”，但最底层都是同一种矩阵乘法。

B · 一根光谱：MLP → LCN → Tiled → CNN

把"受限"做成一个连续旋钮 —— 两个轴：① 要不要局部（核多大）？② 共享多少（G）？

反过来想：MLP 就是“核 = 整张图”的卷积

既然"卷积是受限的 MLP"，那反过来呢？一个全连接层，严格等于把卷积核放大到和输入一样大、只剩一个窗口（M=1）的卷积 —— 不是"类似"，是逐项相等：Linear(N→C) 与 Conv(核=整图) 的权重一一对应。

原因：CNN 的两个约束（局部 + 共享）会随核变大而同时失效 —— 核盖住整图，"局部"就等于"全局"；只剩一个窗口，就没有别的位置可"共享"。两个开关一起关掉，每个输出就连到了全部像素 = MLP。拖动核大小 k 同步看两张图：① 卷积 输入 ✲ 核 = 输出（核长大→输出从 5×5=25 个神经元塌成 1 个、感受野涨满整图）；② 它等价的 MLP 权重矩阵 W——直接回答"一个 k×k 卷积写成全连接到底长什么样"。核越大，W 越稠密；k=5 时 W 塌成一行 25 个独立权重，就是 MLP：

（全 2D。①里绿=输入/输出神经元、蓝框=当前输出感受野、蓝=核；②里每行是一个输出神经元的全部 25 个权重，蓝=非零（各行共享同一套核）、暗=0。这张 W 就是第①节"等价全连接"的真身。）

所以"谁是谁的特例"别用 X⊂Y 硬套，而是一根连续的轴：
核 1×1 —— 最受限的 CNN（局部+共享最强）—— …… —— 核 = 整图 完全自由 = MLP。
· 小核滑动卷积（M>1、真共享）是 MLP 的严格子集 —— 这种卷积 MLP 写不成；
· 整图核卷积（M=1）与 MLP 完全重合。
于是"卷积是受限 MLP"和"MLP 是整图核 CNN"两句都对，只是指着这根轴的两端，并不矛盾。
这不只是理论： 把全连接层重写成"核 = 特征图大小"的卷积，正是 全卷积网络 FCN（Long 等, 2015）和 OverFeat 的核心技巧 —— 换成卷积后再喂进更大的图，它会自动滑动、输出一张类别热力图而不是单个向量，直接拿去做分割 / 检测。

从 MLP 到 CNN 的完整演化：MLP → LCN → Tiled → 卷积

读法 —— 这就是从 MLP 到 CNN 的演化:
· MLP → LCN:加局部(连整图 → 只连窗口)。参数 36 → 16,但每个权重仍只被 N 训练(每个只出现 1 次)。
· LCN → Tiled → 卷积:逐步共享(不同核数 G:4 → 2 → 1)。参数 16 → 8 → 4,同一个权重出现 1 → 2 → 4 次 → 被 1×N → 2×N → 4×N 个 patch 训练(点权重 a 看高亮个数)。
一根旋钮 G(不同核个数)统一全程:每个核的有效样本 ≈ (M/G)×N。G 越小 → 共享越多 → 样本放大越大。MLP 是"还没局部",LCN 是"局部但 G=M 不共享",CNN 是"局部 + G=1 共享到顶"。切分是机会,共享是兑现。
LCN 是谁、什么时候提出的? 「局部连接但不共享」(locally-connected layer,又叫 unshared convolution)没有单一发明人——它是夹在全连接与卷积之间的自然变体。源头是 Hubel & Wiesel(1962) 在猫视觉皮层发现的局部感受野;"共享 vs 不共享"的区分在 LeCun 的卷积网框架里就已内含,后被规范成"locally connected / unshared convolution"(见《Deep Learning》2016 第 9 章)。
它最有名的实战 —— DeepFace(Taigman 等,Facebook,CVPR 2014),效果到底如何?
先把人脸做 3D 对齐(正脸化),于是每张脸的眼/鼻/嘴都落在固定像素位置——同一位置统计稳定、不同位置差别很大。这时卷积"同一个滤波器到处都好用"的假设反而浪费容量;于是 DeepFace 在中间几层改用 LCN,让每个位置学自己的滤波器(就是上图 LCN 那种)。
结果:在 LFW 人脸验证基准上达到 97.35%,把当时最好成绩的错误率砍掉约 27%,逼近人类水平(~97.53%)——这是当年的轰动结果。
但有两个前提,缺一不可:① 人脸已对齐(位置才携带固定语义,不共享才划算);② 有 440 万张标注人脸,撑得住 LCN 暴涨的参数(否则立刻过拟合)。
所以这不是反对共享,而是恰好划出共享的边界:当"位置"本身固定且有语义、数据又足够多时,不共享反而更好;反之(位置会平移、数据有限),共享(卷积)才对。

至此闭环 —— MLP —(加局部)→ LCN —(周期共享)→ Tiled —(全共享)→ CNN。一条轴、两个旋钮(要不要局部？共享多少 G？),就把全连接平滑地演化成了卷积。

C · 局部 ＋ 共享，到底换来什么

直觉上”加限制”像是坏事——表达力更弱了。但对图像来说，这两条限制恰好是对的先验，于是换来四个全连接给不了的好处。

① 参数量：全连接随图像暴涨，卷积恒定不变

”摊平像素”最致命的问题：全连接的参数量 = 输入像素数 × 输出神经元数，图越大参数越离谱。卷积的参数量只取决于核大小 × 通道 × 核数量，和图像尺寸无关——同一个核能扫任意大的图。拿一张标准 224×224×3 的彩色图、各接一层做对比：

方案参数量公式224×224×3 图（输出到 1000 单元）
摊平 + 全连接像素数 × 输出数150,528 × 1000 ≈ 1.5 亿（仅一层！）
一个卷积层核大小 × 通道 × 核数64 × (3×3×3) ≈ 1,792

② 平移等变性：学一次，全图通用

在全连接里，连接”左上角像素”的权重和连接”右下角像素”的权重是完全独立、各学各的。这意味着：哪怕网络在左上角学会了”检测一只眼睛”，它对右下角的同一只眼睛一无所知——得用另一批权重、另一批训练样本，从头再学一遍每个位置。
卷积因为权重共享，同一个核滑遍全图：边缘 / 眼睛检测器只学一次，就在任何位置自动生效。物体平移一下，特征图也跟着平移——这正符合图像的本质（一只猫不会因为挪到右边就变成别的东西）。

③ 保留空间结构：摊平会把”谁挨着谁”丢掉

一旦 flatten 成一长串，像素 (0,0) 和它的邻居 (0,1)、(1,0) 就变成了下标 0、1、4 —— 在全连接眼里，下标 0 和下标 1 的关系，跟下标 0 和下标 500 的关系没有任何区别。也就是说，”哪些像素在空间上相邻”这件事，全连接得从数据里硬学；而卷积一开始就把”邻近像素该一起看”刻进了结构里，白送的。

④ 数据效率与泛化：正确的先验 = 少参数、少数据、不易过拟合

②③ 这两条（局部性 + 平移共享）是对自然图像正确的归纳偏置。先验对了，就能用更少的参数、更少的数据，得到更好的泛化、更不容易过拟合。理论上全连接是 CNN 的超集、能表达同样的函数，但它得用海量数据，自己重新发现卷积一开始就假设好的那些结构。

公平地说，这是一把双刃剑： 卷积的好处全来自”限制”，所以当任务没有平移 / 局部结构时，这个先验反而是包袱。这也是为什么在数据和算力足够大时，限制更少的模型（如 Vision Transformer）能靠”从数据里自己学结构”反超 CNN——少偏置 + 大数据 vs 强偏置 + 小数据，是一组经典权衡。
另外补一个精确说法：卷积层本身是平移等变（输入平移→输出跟着平移），真正的平移不变（输入平移→输出不变）是靠后面的 pooling / 全局平均池化补上的。

一句话收束：摊平 + 全连接是"什么都能学，但什么都得从零学"；CNN是"先把图像的两条物理常识（局部性 + 平移共享）告诉它，于是用极少参数就学得又快又稳"。

D · 多通道怎么算：C_in、C_out 与跨通道求和

你的"切块 → 过 MLP → 拼回去"直觉对了八成。补三个修正，再说清通道维度。

先把那句话精确化（三处修正）

常见说法更精确的说法
划分成多个子图像在每个位置抠出一个窗口，窗口之间会重叠(stride=1时)，不是不相交的切块 —— 这正是 im2col 要复制数据的原因
每块过一个 1×n 的 MLP过的是同一个、共享的 MLP；而且通常有 C_out 个输出通道，所以是 C_out×n 的 MLP，不是 1×n(除非只有 1 个卷积核)
拼回二维矩阵拼回 H_out × W_out × C_out 的三维张量 —— 每个位置吐出的是一个 C_out 维向量，不是单个数

一句话精确版：在每个(重叠)窗口上，用同一个共享的 Linear(C_in·kH·kW → C_out)，再把结果按空间位置摆回去。

卷积核怎么扫多通道输入？（C_in / C_out 的来历）

先看真实的卷积过程，再回到矩阵。设输入有 3 个通道(C_in=3，就像 RGB)，那么一个 filter 内部就有 3 个 2×2 子核——每个子核盯一个输入通道。窗口停在某处时：每个通道各自 ⊙ 求和，再把所有通道的结果加到一起，得到输出的一个数。点"滑动一格"，看这个跨通道相加：

关键就在那个 跨通道相加：一个 filter 不管输入有多少通道，最后都塌成一个数 → 一张输出图（一个输出通道）。两个维度的来历就清楚了：

  • 输入通道数 = C_in → 决定每个 filter 内部配几个子核（图里 2 个），它们被跨通道求和后"看不见"了。
  • filter 的个数 = C_out → 决定输出有几个通道。换一组权重不同的 filter，就多扫出一张输出图：

↑ 同一份 3 通道输入，3 个不同 filter → 3 张输出图，堆成 H_out × W_out × C_out=3 的输出张量

回到矩阵：把"位置 (0,0)"摊平

沿用上面同一份输入和同样 3 个 filter，我们就取卷积过程里 位置 (0,0) 的那个窗口（含 3 个通道）。分四步：① 取窗口 → ② 把 3 个通道的窗口拉平成向量 x → ③ 把 3 个 filter 各自拉平成 W 的 3 行 → ④ 算 W·x = y。最后你会看到 y = [7, 6, 7] 正好等于上面 3 张输出图在 (0,0) 的值——前后对得上：

维度是什么在"小 MLP"里对应
C_in 输入特征维输入图/特征图的通道数。每个核内部有 C_in 个 2D 子核，同时看全部输入通道再加权求和MLP 输入维的一部分(n = C_in·kH·kW)
kH × kW 空间核感受野大小(看多大一块)，不是特征维度输入维的另一部分
C_out 输出特征维卷积核的个数 = 输出通道数。每个核检测一种特征、产出一个通道MLP 的输出维度(有多少个"神经元")
具体例子： RGB 图进第一层卷积，C_in=3；用 64 个 3×3 核，则 C_out=64，权重形状 (64,3,3,3)，每个核 3×3×3=27 个权重。等价于在每个像素位置共享一个 Linear(27 → 64)：输入维 27、输出维 64。特征维度从 3 升到 64。
最干净的特例 —— 1×1 卷积： 当 kH=kW=1，n = C_in，每个位置就是一个不折不扣的 Linear(C_in → C_out)，逐像素独立过同一个 MLP。这正是"每个子图(这里=单像素)过同一个 MLP"的最纯粹形态：输入通道→MLP 输入维，卷积核个数→MLP 输出维，一目了然。

那"RGB 是 3×H×W、是不是更像 MLP 了"？——对，但只在通道维

有了多通道（RGB=3 通道），确实更像 MLP 了，但"像"只发生在通道这一维。看完整求和式就清楚了：

output[co,i,j] = Σci,a,b　input[ci, i+a, j+b] · W[co, ci, a, b]
维度怎么处理像什么
通道 ci每个 filter 对全部输入通道求和（稠密、全连接）MLP
空间 (a,b)只看 kH×kW 局部，且每个位置共享同一套 W卷积

所以一句话：卷积 = 通道维上是 MLP（稠密混合），空间维上是卷积（局部 + 共享）。RGB 只是把"通道维的 MLP"激活了（灰度图时它是平凡的），但空间维仍是卷积。

关键别混："摊平窗口" ≠ "摊平整图"。
· 卷积摊平的是每个局部窗口（含所有通道 C_in·kH·kW），所有位置共享同一个 MLP —— 保留了空间局部性。
· 纯 MLP摊平的是整张 3×H×W 图成一根巨向量，接一个不共享的大 W —— 丢掉空间结构。
RGB 是 3×H×W 只让通道维变稠密，并没有让空间维也摊平。卷积始终是"摊平窗口、滑动复用"，不是"摊平整图"。1×1 卷积就是把空间那一半砍掉、只剩通道 MLP → 退化成逐像素纯 MLP。

落到地面：RGB→灰度，就是一个写死权重的 1×1 卷积

前面说"1×1 卷积 = 逐像素的通道 MLP"。最日常的例子就是 RGB 转灰度：

灰度 = 0.299·R + 0.587·G + 0.114·B

这就是一个 C_in=3 → C_out=1 的 1×1 卷积，核权重 = (0.299, 0.587, 0.114)。灰度转换不是"像"卷积，它本身就是卷积的一个特例——只不过权重写死、且和为 1（保亮度的加权平均）。点"看下一个像素"，看每个像素怎么被三通道加权合成：

所以"CNN 处理 RGB 是把通道相加"这句话，精确版是 加权相加（每通道先乘自己的核权重再求和），和灰度公式是同一个操作，只差三点：

为什么是「相加」？——相加的家谱

时间轴讲了"什么时候用"；这张家谱讲"为什么是相加"。它一路继承自神经元的加权求和，根子在"线性"的定义本身——而卷积的空间求和另有一条信号处理的根：

所以这个"相加" = 矩阵乘法里的 Σₖ = 内积。它不是随便选的——四条"道理"：

① 想要线性，就只能相加

骨架是"线性混合 + 逐点非线性"。线性 ⟺ 加权求和，求和是被逼出来的，不是随便选的。

② 加权求和 = 匹配滤波

⟨w,x⟩ 衡量输入与模板的对齐度，求和把逐点吻合汇成总分。匹配滤波器是噪声中检测已知模式的最优线性检测器。

③ 相加 = 累积证据

每个加权输入是一票证据，神经元把证据加起来、够了就激活。可微、可叠加、数值稳定。

④ 对通道顺序无关

通道没有天然先后，求和（可交换、可结合）是最自然的对称聚合——换通道顺序结果不变。

一句话：这个"相加" = 矩阵乘法的内层 Σ = 内积 = 线性的定义本身。它一头连着 1943 年把神经元建模成"突触求和"的生物直觉，一头连着信号处理里"线性时不变 ⇒ 卷积"的定理。之所以是相加而非别的：我们要让混合保持线性、把非线性单独交给激活函数——而线性，按定义，就是加权求和。

MLP 的"加" vs CNN 的"加"——其实是同一个

常有个误会：以为 MLP 是"加权求和"、CNN 是"直接相加"。其实两者都是同样的两步：先 乘权重（权重在这一步进入），再 平加（系数为 1）。下面用同一套配色标出来——黄色的 + 就是"平加"，它在 MLP 和 CNN 里是同一个东西：

绿 = 输入 x 蓝 = 权重 w 黄 + = 平加（系数 1，到处一样）

MLP 的加权求和

y = w₀×x₀ + w₁×x₁ + w₂×x₂ + …

每项 蓝×绿 = 加权；再用 + 平加。

CNN 的跨通道求和

out = ( w×x+w×x+… )通道0 + ( … )通道1 + ( … )通道2

每个括号内部还是 蓝×绿 的加权和；通道间那个 + 和左边 MLP 的 + 一模一样。

所以"加"是同一个：两边都 = 一个点积 Σₖ wₖ xₖ。× 那一步把权重乘进去了，所以外层的 + 看起来"无权"——其实它本来就该是系数 1 的平加，MLP、CNN 完全一样。CNN 只是把这同一个 Σ 按通道加了层括号而已。真正"无权直接相加"的地方是残差连接 out = F(x) + x，那是另一回事。

★ 全篇收尾：四种网络的权重各装什么知识

底层都是同一种矩阵乘法;装的"知识种类"之所以不同,是因为给权重加了不同的结构约束 —— 沿什么轴复用 + 比较谁。

其实是"两层知识"

知识的两层是什么四种网络
① 显式学到的
(activations × weights 点积)		模板 / 方向 / 关系 / 转移规则四者都有,本质都是"和谁对齐"
