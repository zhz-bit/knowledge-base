Deep Technical Briefing

Transformer 的数学原理：从注意力矩阵到大模型架构谱系

Transformer 的核心不是“某个大模型”，而是一套把序列表示为矩阵、用内容相关的权重进行信息路由、再通过残差堆叠形成可扩展函数逼近器的架构。它的成功来自三件事：全局依赖建模、GPU 友好的矩阵乘法、以及预训练目标与规模化训练的匹配。

Self-Attention Multi-Head Positional Encoding Encoder / Decoder Long Context Scaling Laws MoE / RLHF / Reasoning

一句话主线

Self-attention 把每个 token 同所有 token 的相似度组织成注意力矩阵；多头机制让不同子空间学习不同关系；位置编码补上序列顺序；残差和归一化让这种操作可以堆到几十、上百层。

讲解时的关键提醒

注意力权重是动态计算的“信息路由”，不是固定参数；参数在 Wq、Wk、Wv、Wo、FFN 和归一化中，注意力矩阵 A 随输入变化。

适合深讲的分界线

先讲清楚矩阵形状与 softmax，再讨论位置编码、mask、KV cache、长上下文复杂度。不要一开始陷入模型名字。

数学核心

给定长度 n、隐藏维度 d 的输入矩阵 X ∈ Rn×d，Transformer 的核心是把 X 投影成查询 Q、键 K、值 V，然后用 QKT 构造 token 间的相似度图。

1. 线性投影

Q = XWq, K = XWk, V = XWv

Wq、Wk、Wv 是可学习参数。Q 表示“我要找什么”，K 表示“我能被怎样匹配”，V 表示“被选中后提供什么信息”。

2. 缩放点积注意力

Attention(Q,K,V) = softmax(QKᵀ / √dₖ + B + M) V

B 可用于位置偏置，M 可用于 padding 或 causal mask。除以 √dₖ 是为了控制点积方差，避免 softmax 太早饱和。

3. 多头注意力

headᵢ = Attention(XWqᵢ, XWkᵢ, XWvᵢ) MHA(X) = Concat(head₁,...,headₕ)Wo

多个 head 并不是简单重复，而是在不同投影子空间中学习依存、指代、局部结构、长距离关系等模式。

4. 复杂度

Full attention: time O(n²d), memory O(n²) FFN: time O(nd d_ff)

当 n 很大时，注意力矩阵的 n² 成为瓶颈；当模型宽度很大时，FFN 通常承担大量参数与计算。

Transformer Block 如何工作

AAttention 层

跨 token 混合信息。Encoder 使用双向全注意力；Decoder 常用 causal mask，只能看当前位置之前的 token；Encoder-Decoder 结构在解码器中加入 cross-attention。

FFeed Forward 层

逐 token 的非线性变换。常见形式是 Linear → GELU/SwiGLU → Linear。注意力负责“从哪里取信息”，FFN 负责“如何加工信息”。

RResidual + Norm

残差提供梯度通路，LayerNorm/RMSNorm 稳定激活尺度。现代大模型多采用 Pre-LN 或 RMSNorm，以便训练更深的网络。

原始 Post-LN 风格

u = LayerNorm(x + MHA(x)) y = LayerNorm(u + FFN(u))

原始 Transformer 论文采用这一路径，并配合 warmup、dropout、label smoothing 等训练技巧。

现代 Pre-LN 风格

u = x + MHA(LayerNorm(x)) y = u + FFN(LayerNorm(u))

Pre-LN 通常更易训练深层模型；很多 LLM 还把 LayerNorm 换为 RMSNorm，并使用 SwiGLU FFN。

Transformer 的类别体系

分类不能只按模型名字。更稳妥的方式是按架构、注意力模式、位置处理、训练目标、稀疏性和模态来分。

EEncoder-only

代表：BERT、RoBERTa、ViT。适合理解、分类、检索、表示学习。典型目标是 MLM 或监督分类。

DDecoder-only

代表：GPT、LLaMA、Mistral/Mixtral。适合自回归生成。核心目标是 next-token prediction，推理时使用 KV cache。

SEncoder-Decoder

代表：原始 Transformer、T5、BART。适合条件生成、翻译、摘要。Encoder 理解输入，Decoder 生成输出。

EEncoder-only：双向理解型 Transformer

BERT / RoBERTa / ELECTRA / ViT encoder
输入 tokens 或 image patches
↓
Token/Patch Embedding + Position Encoding
↓
N × Encoder Block
双向 Multi-Head Self-Attention
FFN + Residual + Norm
↓
[CLS] / mean pooling / 每 token 表示
↓
分类、标注、检索或表示学习 Head

结构关键：没有 causal mask，每个 token 可以同时看左侧和右侧上下文；输出通常不是逐步生成，而是一个向量或一组上下文表示。

优势
  • 双向上下文充分，适合理解和判别。
  • 整段输入一次前向，训练和推理并行度高。
  • 输出表示稳定，便于做 embedding、检索和 rerank。
适用场景
  • 文本分类、情感分析、意图识别。
  • NER、抽取式 QA、语义匹配、向量检索。
  • 图像分类、视觉表征预训练。
不适合
  • 开放式长文本生成。
  • 逐 token 流式输出。
  • 需要严格自回归概率分解的任务。

DDecoder-only：自回归生成型 Transformer

GPT / LLaMA / Mistral / Qwen / Mixtral
Prompt tokens
↓
Token Embedding + RoPE/Position
↓
N × Decoder Block
Causal Self-Attention + KV Cache
FFN/SwiGLU 或 MoE + Residual + Norm
↓
LM Head → next-token distribution
↺
把新 token 拼回上下文继续生成

结构关键：使用下三角 causal mask，位置 t 只能看 x<t；推理分为 prefill 和 decode，历史 K/V 会被缓存以降低重复计算。

优势
  • 预训练目标与生成推理一致。
  • 规模化路径简单，容易形成 in-context learning。
  • 统一支持问答、代码、对话、工具调用等文本接口。
适用场景
  • 聊天助手、代码生成、长文写作。
  • 复杂推理、Agent、工具调用编排。
  • 少样本泛化和通用生成任务。
代价
  • 生成必须逐 token 进行，延迟受序列长度影响。
  • 无法在同一层直接看到未来 token。
  • 长上下文下 KV cache 显存压力大。

SEncoder-Decoder：条件生成型 Transformer

Original Transformer / T5 / BART / mT5
Source input: x
↓
Encoder: 双向 Self-Attention → Memory H
↓
Decoder Block
Target prefix: causal self-attention
Cross-Attention: Q 来自 decoder, K/V 来自 H
FFN + Residual + Norm
↓
LM Head → y₁, y₂, ...

结构关键：输入端可以被完整双向编码，输出端仍按自回归生成；cross-attention 是“解码器按当前生成状态查询源输入记忆”。

优势
  • 把“理解输入”和“生成输出”显式分离。
  • 适合强条件任务，源文本不会被因果 mask 限制。
  • cross-attention 提供清晰的源到目标对齐通路。
适用场景
  • 机器翻译、摘要、改写、语法纠错。
  • ASR/OCR 后处理、文档到结构化数据。
  • Text-to-SQL、指令式输入输出转换。
代价
  • 结构比 decoder-only 更复杂。
  • 需要维护 encoder memory 和 cross-attention。
  • 对开放式通用对话不是当前主流基座选择。

Full Bidirectional Attention 每层构造完整 n×n 注意力矩阵，所有 token 双向互看。 信息交互最充分，适合理解任务。 BERT 类文本理解、ViT 图像分类、reranker。
Causal Attention 完整注意力矩阵被下三角 mask 约束，只允许看历史 token。 符合语言生成的概率分解 p(xₜ|x<t)，可缓存 K/V。 GPT 类 LLM、代码生成、对话、流式生成。

Sparse Global Attention 局部窗口 + 少量全局 token + 随机/块稀疏连接。 保留长距离通信路径，同时降低 n² 成本。 长文档分类、长上下文 QA、法律/科研文档分析。
Linear / Kernel Attention 用特征映射 φ 把 softmax attention 近似成 φ(Q)(φ(K)ᵀV)。 理论复杂度可降到 O(n)，适合极长序列探索。 长序列建模、时间序列、资源受限推理；需要接受近似误差。

Vision Transformer 图像切成 patch → 线性投影成 token → 位置编码 → Encoder stack → CLS/Pooling head。 统一用 token 处理图像，全局建模能力强，和自监督预训练兼容。 图像分类、特征抽取、视觉骨干、遥感/医学影像。

讲解时的分类原则

这些类别不是互斥标签，而是可组合的设计轴。例如 Mixtral 是 decoder-only + causal attention + sparse MoE；Swin 是 vision transformer + window attention；T5 是 encoder-decoder + text-to-text 训练目标。介绍具体模型时，最好同时说明它在“架构方向、注意力图、位置机制、训练目标、模态、参数激活”这几个轴上的选择。

发展历程

从 seq2seq 的固定向量瓶颈，到注意力机制，再到可并行的全注意力架构，Transformer 的扩展路径基本沿着“更大规模、更长上下文、更多模态、更强后训练”展开。

2014
Seq2Seq 与注意力前夜

Sutskever 等提出 LSTM seq2seq；Bahdanau 等提出 encoder-decoder 注意力，缓解固定长度向量瓶颈，为 Transformer 的 attention 思想奠基。

2015
Global / Local Attention

Luong attention 系统化比较全局与局部注意力，证明注意力在神经机器翻译中的实用价值。

2017
Attention Is All You Need

Vaswani 等用 self-attention + multi-head + positional encoding 替代循环和卷积，提出原始 encoder-decoder Transformer。

2018
预训练范式分叉：GPT 与 BERT

GPT 采用 decoder-only 自回归预训练再微调；BERT 采用 encoder-only 双向 MLM 预训练，开启大规模语言表示学习。

2019
长依赖与统一文本到文本

Transformer-XL 用 segment recurrence 缓解固定上下文；T5 把多种 NLP 任务统一成 text-to-text；GPT-2 展示零样本/多任务迹象。

2020
规模化、视觉化、长序列化

GPT-3 推动 few-shot；ViT 把图像 patch 当 token；DETR 把检测改成集合预测；Reformer、Longformer、Linformer、BigBird、Performer 探索高效注意力。

2021
多模态、层级视觉与位置编码

CLIP 用图文对比学习连接视觉与文本；Swin 用 shifted windows 成为通用视觉骨干；RoPE、ALiBi 等位置机制影响后续长上下文 LLM。

2022
对齐、计算最优与系统优化

InstructGPT/RLHF 让模型更贴近用户意图；Chinchilla 重新校准参数量与训练 token 的比例；FlashAttention 从 IO 角度优化精确注意力。

2023
开放权重与多模态大模型

LLaMA 证明高质量公开数据和训练配方可得到强基座模型；GPT-4 报告强调 Transformer 预训练、后训练和多模态能力。

2024
稀疏专家与高效推理成为主线

Mixtral 展示稀疏 MoE 在开源生态中的影响；GQA/MQA、KV cache 压缩、量化、推测解码等技术围绕推理成本展开。

2025+
推理后训练与 test-time compute

DeepSeek-R1 等工作凸显大规模 RL 与推理行为塑造。趋势从“只扩大预训练”转向“预训练 + 后训练 + 推理时计算”的组合优化。

位置编码：为什么注意力本身不知道顺序

纯 attention 对输入 token 的排列近似置换等变：如果没有位置机制，它只知道内容相似度，不知道“第几个”。位置编码就是给模型注入顺序归纳偏置。

Sinusoidal 绝对位置

PE(pos,2i) = sin(pos / 10000^(2i/d)) PE(pos,2i+1) = cos(pos / 10000^(2i/d))

原始 Transformer 使用正弦/余弦位置编码。优势是可外推到未见位置；缺点是长上下文泛化不总是稳定。

Relative Position Bias

A = softmax(QKᵀ / √dₖ + b(i - j))

将相对距离作为 attention score 的偏置，直接影响 token 之间的连接强弱。T5 等模型采用类似思想。

RoPE

RoPE(x, m) = R(mθ)x ⟨R(mθ)q, R(nθ)k⟩ depends on (m - n)

把二维子空间中的向量按位置旋转，使点积天然包含相对位置信息。现代 decoder-only LLM 中非常常见。

ALiBi

score(i,j) = qᵢ·kⱼ / √dₖ - slope · |i - j|

不向 embedding 加位置向量，而是在注意力分数中加入线性距离惩罚，常用于 train short, test long 的长度外推讨论。

效率与长上下文

标准注意力的瓶颈是 n²。高效 Transformer 的设计大多是在三件事之间取舍：保留精确 attention、近似 attention、或改变可见 token 图。

路线 核心思想 复杂度变化 代表 适合讲的数学点
稀疏注意力 只让每个 token 看局部窗口、全局 token 或随机连接 从 O(n²) 降到近似 O(n) Longformer、BigBird 把 attention 矩阵视为稀疏图的邻接矩阵
低秩近似 假设注意力矩阵可由低秩投影近似 O(nk)，k ≪ n Linformer 矩阵低秩、投影维度与误差
哈希/聚类 相似 token 分到同一桶，只在桶内计算注意力 O(n log n) Reformer 局部敏感哈希与近似最近邻
核化线性注意力 用特征映射 φ 近似 softmax kernel，利用结合律重排矩阵乘法 O(n) Linear Transformer、Performer φ(Q)(φ(K)ᵀV) 代替 softmax(QKᵀ)V
IO-aware 精确注意力 不改变数学结果，通过 tiling 减少 HBM 与 SRAM 间读写 理论仍 O(n²)，实际显著加速省显存 FlashAttention 计算复杂度之外的 IO 复杂度
KV cache 与 GQA/MQA 自回归推理缓存历史 K/V，减少重复计算；多个 query head 共享 K/V 降低推理常数和缓存占用 PaLM、LLaMA 系列、Mistral 类模型 prefill 与 decode 阶段的复杂度差异

训练目标、损失函数与缩放规律

自回归语言模型

L = - Σₜ log pθ(xₜ | x<t)

Decoder-only LLM 的主目标。训练时可并行计算所有位置的损失；推理时必须逐 token 生成，因此 KV cache 很关键。

Masked Language Model

L = - Σ_{t∈mask} log pθ(xₜ | x_visible)

BERT 类模型使用双向上下文预测被 mask 的 token，更适合理解与表示，不天然适合长文本自回归生成。

Seq2Seq / Denoising

L = - Σₜ log pθ(yₜ | y<t, Encoder(x))

T5/BART 类模型把任务统一为条件生成，适合翻译、摘要、问答等输入到输出的任务。

对比学习

L_CLIP = CE(sim(imageᵢ,textⱼ)/τ, matching pairs)

CLIP 用图像编码器和文本编码器在共同嵌入空间中对齐，为多模态检索、分类和生成模型条件控制提供基础。

缩放规律的讲法

经验 scaling laws 观察到模型性能随参数量 N、数据量 D、计算量 C 增大呈较平滑的幂律改善；Chinchilla 后的重要修正是：在固定计算预算下，参数量和训练 token 数需要更均衡地增长，单纯堆参数会造成“欠训练”。

  • 预训练阶段主要优化 next-token 或 masked-token 的交叉熵。
  • 后训练阶段通过 SFT、偏好优化、RLHF/RLAIF/RL 改变模型输出分布和交互行为。
  • 推理阶段的 test-time compute 会把“训练好的分布”进一步转化为搜索、采样、验证或长链推理过程。

现代相关知识

1MoE

把 FFN 替换为多个专家，通过 router 为每个 token 选择少数专家。总参数很大，但每 token 激活计算相对可控。

2RAG

检索增强不是 Transformer 内部结构，但能把外部知识以上下文 token 形式注入模型，缓解参数记忆与更新成本。

3Multimodal

图像 patch、音频帧、视频片段都可 token 化，再通过 cross-attention 或统一 token 序列与文本对齐。

4Reasoning

推理模型通常不是换掉 Transformer，而是通过数据、RL、搜索、验证器和推理时计算改变行为。

5Alignment

RLHF 的核心是从人类偏好训练 reward model，再优化策略，使模型输出更符合有用、真实、安全等偏好。

6Quantization

降低权重和激活精度以节约显存与带宽。对 Transformer 推理而言，内存带宽和 KV cache 往往与 FLOPs 同样重要。

7Speculative Decoding

用小模型草拟多个 token，再由大模型验证，在保持分布正确的前提下降低自回归延迟。

8替代架构

SSM、RWKV、Mamba 等路线试图用线性时间序列建模替代全注意力，核心目标是长上下文和推理效率。

技术分享建议结构

60 分钟版

  • 5 min：Transformer 解决了 RNN/CNN seq2seq 的什么问题。
  • 15 min：Q/K/V、softmax、mask、多头注意力的矩阵推导。
  • 10 min：Block、残差、LayerNorm、FFN、训练稳定性。
  • 10 min：Encoder-only、Decoder-only、Encoder-Decoder 分类。
  • 10 min：位置编码与长上下文效率。
  • 10 min：BERT/GPT/T5/ViT/CLIP/LLM 的发展线索。

90 分钟深讲版

  • 加入注意力矩阵的图解释：A 是输入相关的稠密有向加权图。
  • 加入复杂度推导：训练、prefill、decode、KV cache 的差异。
  • 加入 RoPE/ALiBi 的位置外推对比。
  • 加入 scaling laws、Chinchilla、FlashAttention、MoE 的工程影响。
  • 最后讨论“Transformer 是架构，LLM 是训练范式和系统工程的结果”。
