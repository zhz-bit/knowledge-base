RESEARCH · 一门手艺

没人真正教你做研究

他们给你一张桌子、一个别人选好的问题,和一句含糊的"做点新东西"。于是多数人逆向工程出"看起来像研究者",而不是"成为研究者"。真正的技能,是一摞更小的技能——而它们几乎每一个,都能被刻意训练。

01

PICK YOUR OWN PROBLEMS自己选问题

别"吸收"问题,要"选择"问题。

Hamming 在贝尔实验室有个让人午餐时换桌子的习惯:问邻座"你领域里重要的问题是什么?",再问"那你为什么不做?"。问题之所以刺人,是因为多数人没有好答案——我们不选问题,我们吸收问题:从导师、从某大厂上季度的发布、从这周被疯狂转发的论文。

吸收来的问题,麻烦在于你拿着结论,却没有推理:你知道某名实验室在乎某方向,却不知道为什么、期待发现什么、什么会让他们放弃。他们一转向,你一年后才知道。而在一个已经时髦的问题上,你在和一千个起步更早、算力更多的人赛跑。

Schulman 把研究分两种模式:① 读文献、找可改进处;② 选一个你真心希望它存在的结果,再反推实验。他主张第二种——安静的理由是:它制造原创性。一个你真在乎的目标,会把你拖进任何综述都没覆盖的领地。

品味(taste)常被当作天赋,它其实更像肌肉:每个实验跑之前先预测结果;遮住论文的结果区,只凭方法猜数字;标记本月发布里哪些两年后还重要,之后核对命中率。一次预测 + 一次校正,重复几百遍——所有好模型就是这么训出来的,包括你脑子里那个。

训练:① 写下"我希望存在的结果",反推实验;② 跑每个实验前先押结果;③ 遮住结果区猜数字;④ 给本月发布打分,日后核对命中率。
📌 Hamming《You and Your Research》· John Schulman《ML Research 指南》
02

UPGRADE YOUR INPUTS升级你的输入

共享的阅读清单,产生共享的(因而几乎无价值的)想法。

如果你的信息食谱是 arXiv 热榜 + 群聊过滤后剩下的东西,你会可靠地和所有人在同一时间得出同一结论——而这种结论值约等于零。

老料被严重低估。这个领域延迟地重跑自己的过去:MoE 可追到 1991、LSTM 到 1997、反向传播 1986 才主流化。Sutton 用大约一千字写下 2019 的《苦涩的教训》,比十倍长的综述更能预测领域的形状。Shannon 1952 谈创造性思维,开场第一招是:把问题缩小到近乎平凡,先解小版本,再一块一块把难度加回来——这一招能带你穿过比任何现代效率建议都多的墙。

广度和深度一样重要。可解释性厚着脸皮借神经科学;eval 设计是穿白大褂的机制设计;真懂 GPU 如何搬内存,能在 benchmark 出来之前就判断哪些架构论文注定失败;而诚实的统计或许是 ML 最稀缺的技能——很多发表出来的"严谨",不过是带误差棒的氛围。

还有一条:读论文本身,别读总结它的线程。附录是埋尸体的地方,而 limitations 段通常是全文最诚实的一段。

训练:① 每周读一篇 5 年以上的老论文;② 用 Shannon 法把当前问题缩到平凡再加难度;③ 跨一个邻近领域(神经科学/机制设计/系统/统计);④ 读原文 + 附录 + limitations,不读线程。
📌 Sutton《The Bitter Lesson》· Shannon《Creative Thinking》1952
03

WRITE EVERYTHING DOWN把一切写下来

写作会逼出你脑子糊弄过去的缝隙。

Paul Graham 指出:一个想法在你试图把它写成文字之前,会一直感觉很完整。落到纸上,缝隙就现形了——那个你从没验证的假设、那个其实不成立的推导、那两个悄悄互相矛盾的论断。

Feynman 的规矩:第一个你必须避免欺骗的人是你自己,因为你是最容易得手的目标。写作是有史以来最便宜的防御。Darwin 走得更远、把它程序化:任何与他理论相悖的事实,当场记下——因为他发现自己的记忆删除不便证据比删除便利的更快。你的记忆对失败的 run 也是这么干的。

所以记日志:假设、设置、预期、结果、更新后的信念。重读上个月的条目,谦卑得任何审稿人都给不了。

然后把其中一部分公开。Olah & Carter 的《Research Debt》论证:领域会被未消化的想法噎住,而一个清晰的解释是真正的贡献,不是打杂。今天很多做可解释性的人,是通过可读的博客、而非会议论文入门的。一批公开写作,还是你能持有的最强凭证——因为它是一份无法伪造的"你如何思考"的样本。

训练:① 实验日志五栏:假设/设置/预期/结果/更新信念;② 当场记下与你想法相悖的证据;③ 每月重读旧日志;④ 把一个难概念用平实语言公开讲清楚。
📌 Paul Graham · Feynman · Darwin · Olah & Carter《Research Debt》
04

TIGHTEN THE LOOP收紧循环

研究速度,基本就是你发现自己错了的速度。

关于 Alec Radford 的故事很少关于灵光一现,而是关于量:每天更多 run、每周丢掉更多错误想法、一个比谁都更新得快的现实模型。这才是真正的游戏。

于是工具是一等的研究活动:启动一个 run 应是一条命令,画它再一条;每个实验都能从它的 config 复现;比较两个 run 应是几秒,而不是一下午的考古。Karpathy 的训练配方里有一步回本一百次:大规模训练前,先在单个 batch 上过拟合。三十秒,一半的 bug 没了。把一切缩到便宜 → 搞对 → 再花算力。

还要退掉"工程是初级伙伴"的念头。在前沿,两份工作已经融合:能亲手搭 harness、eval、data pipeline 的研究者,他的假设才真正得到检验;其他人在排队等着。

训练:① 一条命令启动 run、一条命令出图;② config 即可复现,两个 run 秒级对比;③ 单 batch 过拟合再放大;④ 把"建工具/管线"当研究本身来投入。
📌 Alec Radford(迭代量)· Karpathy《A Recipe for Training Neural Networks》
05

STARE AT THE OUTPUTS盯着输出看

下降的 loss 曲线不是分析,是安慰剂。

你的实验抛出的信息,远多于你消费的:transcripts、失败案例、分布那条奇怪的尾巴——大多没人读,死在 logs 文件夹里。

Karpathy 的配方在写任何训练代码之前,先花数小时手工看原始数据。多数 ML bug 住在数据里,而且静默失败:什么都不崩,你只是得到一个平庸的模型,和一套关于"为什么平庸"的错误理论。

Andrew Ng 把同一个不光彩的动作教了十多年,因为没有什么比它更管用:拉一百个失败案例,全部读完,分成几堆,攻击最大的那堆。它对模型有效,对 eval 也有效——一个你从没读过 transcript 的 benchmark,是一个你并不真懂的 benchmark。一条真正反常行为的 transcript,教你的比下一位小数的准确率多。

训练:① 写训练码前先手翻几小时原始数据;② 误差分析:拉 100 个失败、读完、分堆、攻最大堆;③ 用过的 benchmark 必读其 transcript;④ 专门去看分布的尾巴与反常案例。
📌 Karpathy(先看数据)· Andrew Ng(误差分析)
06

WANDER ON PURPOSE有意识地漫游

你的第一个子领域是时机的意外,就当意外对待。

在 interpretability、evals、rl、systems 里都真实地待一阵,再决定定居哪里。这个领域的某个角落,你那份特定的"怪"会是不公平的优势——而找到它的唯一办法,是在好几个地方交学费。没人给你免单。

每个想法先跑一次性的、可丢弃的版本,让多数早夭。把 baseline 调到痛为止——ML 的坟场堆满了对着一个调好的 baseline 就蒸发的收益,而审稿人是你最不该从他那里学到这件事的人。消融(ablate)到你知道是哪个组件扛起了结果:通常只有一个,而且通常不是标题里那个。

广度也是保险:子领域都会饱和,通常就在它于 Twitter 上达峰之后。能熬过这些转型继续产出的人,是那些早就熟悉邻近领地的人。

训练:① 在 3–4 个子领域各交一次学费再定居;② 想法先做一次性版本,大胆早夭;③ baseline 调到痛、ablate 到锁定真正起作用的组件;④ 平时维护一两块"邻近领地"作保险。
📌 调好的 baseline · 消融 · 广度即保险
07

FIND YOUR PEOPLE找到你的人

关着门的人做得更多,开着门的人做出真正重要的工作。

Hamming 注意到一个模式:关上办公室门的同事,在任一年里都做得更多;而开着门的同事,做出了真正要紧的工作——因为那些打扰携带着"世界究竟需要什么"的信息。你的"门"大概是收件箱,让它开着。

慷慨在研究里的复利无与伦比:复现一个结果并公布你的发现;发布你为自己造的工具;用平实语言解释一件难事。回报会从侧面、几个月后到来——作为那个你申请不到的合作、引用,或职位。也把半成形的想法公开漂浮,因为在时间线上犯错,远比在印刷品上犯错便宜。而那个在你投入三个月之前就告诉你"这想法很烂"的合作者,比算力更值钱——这种关系买不到,只能挣来。

训练:① 保持"开门"(可被打扰的入口);② 复现并公布、发布工具、公开讲清难点;③ 公开漂浮半成形想法,早点被纠错;④ 珍惜那个敢说你"想法烂"的人。
📌 Hamming(开门 vs 关门)· 慷慨的复利
08

THE LONG GAME长期游戏

每日的边际优势会像利息一样复利。

Pasteur 说机会偏爱有准备的头脑;Hamming 在此之上建了整套职业哲学:知识与生产力像利息一样复利。那些每日的小边际——你读什么、记什么、循环跑多快、和谁争论——孤立看都微不足道。给它们几年,就长成从外面看像运气的职业。

比"感觉有必要"更早开始复利。未来的你已经知道:这是便宜的那部分。

📌 Pasteur(有准备的头脑)· Hamming(复利)

贯穿全文的元模式

8 个习惯底下,其实只有这几条引擎在反复出现。

  • 预测 → 校正,重复几百遍:品味、看输出、写日志,本质都是"押一个、对一下、更新信念"。
  • 研究速度 = 发现自己错了的速度:所以收紧反馈环、把一切缩到便宜、单 batch 过拟合。
  • 复利:每日微小边际(读/记/循环/争论),几年后从外面看像运气。早点开始。
  • 别欺骗自己(你最易得手):写作、日志、读原始数据/transcript、调到痛的 baseline、消融——都是防自欺的装置。
  • 原创是"制造"出来的:选一个你希望它存在的目标 + 喂冷门/老料的输入,把你带到综述覆盖不到的地方。
  • 慷慨与开门:复现、发布工具、公开讲清、漂浮半成形想法——回报从侧面、几个月后到来。

金句背后:文章一笔带过、其实有真东西的几条

原文很多是"一句话带过、背后有完整方法"的点。分两层拆透:动手实操的纪律链,与容易被技术细节盖过的判断力。

实操层 · 一条防"沉默 bug"的纪律链

这五步串起来,是一条针对沉默的 bug 的防御链——多数 ML bug 不会崩溃报错,而是静默地让你得到"一个平庸模型 + 一套关于它为什么平庸的错误理论"。

  1. 写训练码前,先肉眼过几小时原始数据。数据 bug 静默失败:标注差一帧、坐标系反了、某类样本全脏——程序照跑、loss 照降,你只得到"平庸模型 + 错误归因"。(Karpathy)
  2. 大规模训练前,先在单个 batch 上过拟合。管线对的话,模型一定能把几条样本背到 loss≈0;背不动 = 梯度没回传 / 标签错位 / loss 写错。三十秒,干掉一半 bug。(Karpathy)
  3. baseline 调到心疼,再做消融。否则你那"几个点"可能只是"方法被你认真调、baseline 被你敷衍"。消融到你知道是哪个组件扛着结果——通常一个,且通常不是标题里那个。
  4. 读 100 个失败案例,分堆,攻最大的那堆。(Andrew Ng 的 error analysis) 汇总数字只说"错多少",不说"为什么错";一条诡异行为的 transcript,比下一位小数的准确率教你更多。没读过 transcript 的 benchmark,是你不真懂的 benchmark。
  5. 读论文时盖住结果、只凭方法猜数字。把"读论文"从被动接收变成带反馈的预测训练——你猜的数与真实数的差距,精确暴露你直觉哪里不准。重复几百次,品味就长出来了。

策略层 · 四条容易被技术细节盖过的判断力

  • 原创性是"反推目标"的副产品,不是硬想的靶子。你真心想让某个结果存在、而它还不存在,那通往它的路上必然有没人走过的段落——你是被问题逼进无人区,而非刻意去找无人区。
  • 投资"判断自己错了的速度",而不是"做对的速度"。你大多数想法是坏的(常态),决定产出的是淘汰坏想法多快:坏想法做到 50% 才放弃,一篇好论文要付三篇的工作量;10% 就判死刑,效率差几倍。而且"在时间线上犯错,远比在印刷品上犯错便宜"——半成形想法尽早暴露受击。
  • 公开写作 = 你能持有的最强、且无法伪造的资历。CV 说你做过什么,博文直接展示你怎么思考,没法注水;而把难事讲清本身就是贡献(帮领域消化"未消化的想法")。
  • "开门" = 获取"什么重要"的信号渠道,不是对深度工作的干扰。关门的人产出更多,开门的人做出重要的工作——因为打扰携带"世界究竟需要什么"的信息。今天你的门大概是收件箱,让它开着。
一句话内核
研究的成败,不取决于你执行好想法的能力,而取决于一组可以刻意训练的判断力——判断什么问题值得做、判断自己什么时候错了、判断一个结果是真的还是假的——而这些判断力,全都通过"预测 + 暴露 + 修正"的高频循环积累,且会像利息一样复利。
